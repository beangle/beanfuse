    认证和权限验证模块。
    主要功能在于认证用户和验证用户访问的资源是否在权限范围内。
    认证接口Authentication,验证接口Authorization,OnLineMonitor是在线运行接口。
    DefaultMonitorImpl是实现这三个接口的缺省实现。
    如果使用该认证框架，需要
    1、实现SystemUser接口，提供几个必要的属性。
    2、实现SystemUserService接口，提供用户查询和权限查询的服务
    3、配置过滤器
	<filter>
		<filter-name>CheckAuthorityFilter</filter-name>
		<filter-class>org.beanfuse.security.support.CheckAuthorityFilter</filter-class>
		<init-param>
			<param-name>loginFailPath</param-name>
			<param-value>index.do</param-value>
		</init-param>
		<init-param>
			<param-name>noAuthorityPath</param-name>
			<param-value>globalError.do?errorCode=notEnoughAuthority</param-value>
		</init-param>
	</filter>
	<filter-mapping>
		<filter-name>CheckAuthorityFilter</filter-name>
		<url-pattern>*.do</url-pattern>
	</filter-mapping>
	4、配置监听器
	<listener id="OnlineUserListener">
		<listener-class>org.beanfuse.security.support.OnlineUserListener</listener-class>
	</listener>
    5、配置online_monitor.properties
    在classpath的根路径下配置online_monitor.properties
	#用户种类
	monitor.user.categories=1,2,3
	#在线用户数限制
	monitor.onLineLimits=200,300,100
	#最大在线不活动时间
	monitor.maxInactiveIntervals=
	#不限制权限的action
	monitor.ignore.actions=
	#是否限制使用系统的最大用户限制
	monitor.isMaxUserLimited=false
	6、配置spring
	由于缺省的安全实现使用了spring，需要在spring中查找OnLineMonitor.MONITOR_NAME,因此需要在spring的applicationcontext中
	配置该bean.
	7、在系统自己的login代码中调用OnLineMonitor的login接口。对于返回的字符串进行处理。如果返回""，表明登录成功。
	其他不成功的字段可以在自己的国际化文件中进行说明，以显示在用户的登录界面上。
